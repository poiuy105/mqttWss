# 开机自启动问题排查指南

## 问题现象
设备重启后，应用未能自动启动。

## 可能原因及解决方案

### 1. Android 系统限制（最常见）
Android 10+ 对后台启动 Activity 有严格限制，需要特殊权限。

**解决方案：**
- 在设置页面点击"🔋 电池优化"按钮，选择"允许"或"不优化"
- 在设置页面点击"⚡ 极速模式白名单"按钮（比亚迪车机专用）
- 在设置页面点击"🔄 自启动管理"按钮，确保应用被允许自启动

### 2. 缺少必要配置
BootReceiver 需要保存的 MQTT 配置才能自动连接。

**解决方案：**
- 确保已在连接页面填写并保存了 MQTT 配置（主机、端口、Client ID 等）
- 确认配置已正确保存（重启应用后配置仍然存在）

### 3. 无障碍服务未启用
如果依赖无障碍服务，需要确保其已启用。

**解决方案：**
- 在设置页面点击"📱 ADB 无障碍锁定"按钮，按照指引操作
- 或在系统设置 → 无障碍中手动启用 MQTT Assistant

### 4. 厂商特定限制
不同厂商有不同的电源管理和自启动管理机制。

**比亚迪车机特别注意事项：**
- 必须配置三项白名单：极速模式、电池优化、自启动
- 建议使用 ADB 命令永久化无障碍服务配置
- 部分车型可能需要额外的系统级授权

## 验证步骤

1. **检查当前状态：**
   ```bash
   adb shell pidof io.emqx.mqtt
   ```
   如果返回进程ID，说明应用正在运行。

2. **检查自启动权限：**
   - 进入系统设置 → 应用 → MQTT Assistant
   - 查看是否有"自启动"选项并已开启

3. **测试重启：**
   - 完全关闭应用
   - 重启设备
   - 观察应用是否自动启动

## ADB 命令辅助

### 检查应用是否收到 BOOT_COMPLETED 广播
```bash
adb logcat | grep BootReceiver
```

### 手动触发 BOOT_COMPLETED 广播（测试用）
```bash
adb shell am broadcast -a android.intent.action.BOOT_COMPLETED -p io.emqx.mqtt
```

### 检查电池优化状态
```bash
adb shell dumpsys deviceidle whitelist | grep io.emqx.mqtt
```

## 代码层面的改进

本次更新包含以下改进：

1. **移除了 autoStart 标志依赖**
   - 之前：只有当 `autoStart=true` 且有配置时才启动
   - 现在：只要有保存的配置就尝试启动

2. **添加了 REQUEST_IGNORE_BATTERY_OPTIMIZATIONS 权限**
   - 允许应用请求忽略电池优化

3. **新增 BootUtils 工具类**
   - 提供统一的自启动检查和设置功能
   - 支持多种厂商的自启动管理页面

4. **增强了日志记录**
   - BootReceiver 中添加了详细的日志输出
   - 便于排查启动失败的原因

## 推荐操作流程

1. 首次安装后，先手动打开应用并完成 MQTT 配置
2. 在设置页面配置所有保活选项：
   - ✅ ADB 无障碍锁定（按指引操作）
   - ✅ 极速模式白名单
   - ✅ 电池优化（设为允许）
   - ✅ 自启动管理（设为允许）
3. 重启设备测试
4. 如仍无法启动，查看 logcat 日志排查具体原因

## 常见问题

**Q: 为什么有时能自启动，有时不能？**
A: 可能是系统资源紧张时杀死了后台进程，或者电源管理策略临时变更。建议配置所有保活选项。

**Q: ADB 命令执行后重启又失效了？**
A: 某些系统会在重启时重置安全设置。确保执行了完整的 ADB 命令序列，包括 pm grant 权限授予。

**Q: 能否完全不依赖用户操作实现自启动？**
A: 在较新版本的 Android 上几乎不可能。系统出于安全和性能考虑，要求用户明确授权。
